# Plan: AST-Based Code Validation (#49)

## Overview

A validation pipeline that intercepts all code (YAML manifests, kubectl commands) before it reaches the cluster — whether generated by AI, edited by the user, or applied from the YAML tab.

## Core Concept

Instead of regex-based validation, we parse code into an Abstract Syntax Tree (AST) and validate the tree structure. This catches errors that regex misses (nested structures, context-dependent rules).

## Integration Points

### 1. YAML Tab Editor

When user clicks "Apply" in the YAML tab:

```
User clicks Apply
  → validateYaml(content)
  → if errors: show inline, block Apply
  → if warnings: show inline, allow Apply with warning
  → if clean: proceed to kubectl apply
```

### 2. AI Chat Responses

When `turn_complete` arrives:

```
AI response received
  → extractCodeBlocks(response)
  → validate each block
  → render with validation badges
  → Run/Copy buttons only for validated blocks
```

### 3. Edit YAML Panel

When editing any resource's YAML:

```
User edits in textarea
  → debounce 500ms
  → validateYaml(content)
  → show validation results below editor
  → Apply button shows error/warning count
```

## Implementation Sequence

1. **YAML AST Parser** (2 days)
   - parseDocument wrapper with error collection
   - K8s schema rules (required fields, value formats)
   - Best practice rules (security, resources, probes)
   - Tests for each rule

2. **kubectl Command Tokenizer** (1 day)
   - Tokenizer with quote handling
   - Verb/resource/flag extraction
   - Risk classification
   - Tests

3. **Code Block Extractor** (1 day)
   - Extract from AI responses
   - Route to appropriate validator
   - Aggregate results

4. **UI Integration** (2 days)
   - YAML tab: inline validation results
   - Chat: command cards with risk badges + Run buttons
   - Apply gate: block on errors, warn on warnings
